# ClipTrio Render Smoke Test

This workflow verifies the PC Web V1 MP4 render contract with deterministic source videos.

## What It Checks

- Docker renderer at `http://127.0.0.1:3000/api/render`.
- Local development proxy at `http://127.0.0.1:3001/api/render`.
- HTTP success, `video/mp4`, and non-empty output.
- H.264 video, AAC audio, 1080 x 1920, 30fps, yuv420p, and expected duration.
- Top, middle, and bottom source order.
- Different source start times.
- Two-second segment looping across a four-second export.
- Caption-enabled render path.
- A 1080 x 1920 review PNG extracted from each result.

Generated fixtures and outputs are written under `tmp/`, which is ignored by Git.

## Prerequisites

Start the Docker renderer:

```powershell
docker compose up -d --build
```

Start the local UI with proxy rendering in another terminal:

```powershell
$env:PORT='3001'
$env:RENDER_PROXY_URL='http://127.0.0.1:3000/api/render'
npm start
```

The scripts use local FFmpeg and FFprobe when available. Otherwise, they automatically use the running `clip-trio` Docker Compose service.

## Commands

Generate or reuse the three deterministic source videos:

```powershell
npm run fixtures
```

Regenerate them from scratch:

```powershell
npm run fixtures -- --force-fixtures
```

Verify both direct and proxy rendering:

```powershell
npm run smoke:render
```

Verify one path only:

```powershell
npm run smoke:render:direct
npm run smoke:render:proxy
```

Optional environment variables:

- `FFMPEG_PATH` and `FFPROBE_PATH`: local executable paths.
- `MEDIA_TOOL_MODE=local`: require local tools.
- `MEDIA_TOOL_MODE=docker`: require Docker tools.
- `MEDIA_DOCKER_SERVICE`: Compose service name, default `clip-trio`.
- `SMOKE_DIRECT_URL`: direct render URL.
- `SMOKE_PROXY_URL`: proxy render URL.
- `SMOKE_RENDER_TIMEOUT_MS`: request timeout, default 180000ms.

## Test Scenario

The fixtures use red, green, and blue color families that change every second. The render request uses:

- Starts: top `0s`, middle `1s`, bottom `2s`.
- Clip length: `2s`.
- Export length: `4s`.
- Captions: `TOP TEST`, `MIDDLE TEST`, and `BOTTOM TEST`.

The script samples each section at `0.25s`, `1.25s`, and `2.25s`. The third sample must match the first, proving that the selected two-second segments looped.

Results are saved as:

```text
tmp/smoke-results/direct.mp4
tmp/smoke-results/direct-frame.png
tmp/smoke-results/proxy.mp4
tmp/smoke-results/proxy-frame.png
```

Open the review PNGs to confirm the three captions visually. Their dimensions are checked automatically.

## Browser PNG Export Check

The product's PNG export is generated by browser Canvas and does not use `/api/render`. Node tests now verify the 1080 x 1920 backing store, three contiguous 1080 x 640 section boundaries, cover geometry, caption placement, and the shared FFmpeg layout contract. The final encoded PNG still needs this deterministic visual browser check:

1. Open `http://127.0.0.1:3001/`.
2. Load the top, middle, and bottom fixture videos from `tmp/smoke-fixtures/`.
3. Set their segment starts to 0s, 1s, and 2s.
4. Set the preview to the start of the selected segments.
5. Select `三拼图片` and export.
6. Confirm a 1080 x 1920 PNG with red, green, and blue sections and all three captions.

Do not commit fixture videos, rendered MP4 files, or extracted PNG files.

Canvas uses the browser's system font stack while MP4 rendering uses Noto Sans CJK in Docker. Glyph rasterization and exact text bounds can therefore differ slightly even though section geometry, caption text, placement parameters, and gradient start are centrally controlled.

## Last Verified Result

Verified on 2026-07-23 during the `v1.0.0` release acceptance:

- 74/74 Node tests passed, including the browser-discovered preview resolution-label fix.
- Syntax checks passed.
- Direct Docker render passed: H.264/AAC, 1080 x 1920, 30fps, yuv420p, 4.00s.
- Local proxy render passed with the same contract.
- Both outputs were 369392 bytes for the deterministic scenario.
- Samples at 0.25s and 2.25s matched, confirming the two-second loop.
- Review frames showed all three sections, captions, and caption gradients.
- Browser acceptance passed at 1440 x 900 and 1280 x 720 with no horizontal overflow and the export action visible.
- Browser PNG exports completed at 1080 x 1920 and 720 x 1280 with correct recent-export metadata.
- Browser MP4 export completed through the local proxy at 720 x 1280 and 60fps.
- A controlled renderer outage produced a persistent actionable error, preserved all sources and settings, and succeeded on retry after the renderer recovered.
- The browser console contained no application errors.

The host's development-snapshot FFmpeg/FFprobe build returned `write EOF` while the smoke script piped media through the Windows process. Running the same verification with `MEDIA_TOOL_MODE=docker` passed. The Docker render request itself also succeeded independently through `curl`, so this was treated as a host media-tool pipe compatibility issue rather than a renderer failure.
